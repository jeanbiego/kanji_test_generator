## タグ作成ルール

### 1. タグ命名規則
- **形式**: `v{メジャー}.{マイナー}.{パッチ}-{状態}-{機能名}`
- **例**: `v1.0.0-stable-basic-functionality`
- **例**: `v1.1.0-feature-pdf-generation`
- **例**: `v1.0.1-fix-validation-error`

### 2. バージョン番号の意味
- **メジャー**: 大きな機能追加や破壊的変更
- **マイナー**: 新機能追加（後方互換性あり）
- **パッチ**: バグ修正や小さな改善

### 3. 状態ラベル
- **stable**: 安定版（本番環境で使用可能）
- **beta**: ベータ版（テスト用）
- **alpha**: アルファ版（開発用）
- **feature**: 機能追加版
- **fix**: バグ修正版

### 4. 機能名
- 実装した機能や修正内容を簡潔に表現
- 例：`pdf-generation`, `validation-error`, `ui-improvement`

## 実装メモとの紐づけルール

### 1. 基本方針
各タグの作業内容と、対応する実装メモファイルを明確に紐づけることで、開発履歴の追跡性を向上させます。

### 2. 実装メモファイルの配置
- **実装中**: `docs/`直下に配置
- **実装完了後**: `docs/implementation/`に移動
- **ファイル名変更禁止**: 実装完了後もファイル名は変更しない

### 3. リンク形式
- **実装中**: `[ファイル名](docs/ファイル名)`
- **完了後**: `[ファイル名](docs/implementation/ファイル名)`

### 4. ファイル命名規則
- **形式**: `{YYYYMMDD}_implementation_{タスク名}.md`
- **例**: `20250127_implementation_basic_functionality.md`
- **例**: `20250127_implementation_pdf_generation.md`

## 日付記入ルール

### 1. 日付確認手順
**重要**: 日付を記入する際は、必ず以下の手順で確認してから記入すること
1. **システム日付の確認**: `Get-Date` または `date` コマンドで現在日付を確認
2. **YYYY-MM-DD形式**: 必ず年-月-日の形式で記入
3. **二重確認**: 記入前に日付の正確性を再確認
4. **誤記防止**: 年号や月日の誤記を防ぐため、慎重に記入

### 2. 日付形式
- **正しい例**: `2025-01-27`
- **間違った例**: `25-01-27`, `2025/01/27`, `2025年1月27日`
