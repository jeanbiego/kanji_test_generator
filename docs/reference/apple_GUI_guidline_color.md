# Apple人間インターフェイスガイドライン — Color

## 色の役割と基本方針
- インタラクティブな要素を示し、ブランドの印象や活力を伝える手段として色を活用する
- Light / Dark モードの両方で視認性と一致性を保つタッチカラー（tint color）を設計に組み込む

## アプリ全体で統一する色使い
- 補色を活用した調和のある色選びを推奨
- アプリのインタラクティブ性を示すキーとなる「基調色（キーコントロールカラー）」をひとつ定める  
  （例：Notes は黄色、Calendar は赤）
- アプリロゴやブランドに沿った限定的なカラーパレットを使用する

## カラー技術仕様
- iOS デフォルトは sRGB カラースペースだが、P3（広色域）をサポートするディスプレイでは Display P3 を使う
  - 特に鮮やかな色彩や画像では、16-bit PNG + Display P3 プロファイルが望ましい
- macOS や iOS のシステムカラーピッカーで P3 と sRGB を比較しながら選定可能

## 色の使い分けと視認性配慮
- インタラクティブ要素と非インタラクティブ要素で同じ色を使わない
- 透過（translucency）による色の見え方への影響を考慮

## 実世界での表示確認
- 屋内外、時間帯、明るさの異なる条件下で色の確認が必須
- True Tone ディスプレイでは、環境光に応じて白点が変化するため  
  特に読書、写真、動画、ゲームなど色に敏感なコンテンツで調整スタイルを指定可能

## アクセシビリティと多様性への配慮
- 色覚異常（色盲）や文化的背景により色の認識には差があるため  
  色だけで意味を伝えず、形状やアイコンなど別の区別手段も併用する
- WCAG 基準に基づいたコントラスト比を確保する  
  - 最低でも 4.5:1、理想は 7:1 以上を目指す
