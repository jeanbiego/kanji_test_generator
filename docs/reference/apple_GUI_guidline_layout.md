# Apple Human Interface Guidelines — Layout

## 基本理念
- レイアウトは、アプリの内容や構造を明確に伝え、ユーザーの理解や操作を助ける重要な要素。
- 一貫性、整列、階層、明瞭性、応答性の5つの原則に基づいて設計する。

## 一貫性
- レイアウトパターンをアプリ全体で統一することで、予測可能性と安心感を提供する。
- システム標準のコンポーネントやマージン、余白設定に従うと一貫性が保ちやすい。

## 整列
- グリッドや基準線に基づく整列により、視覚的な安定感と読みやすさが向上する。
- 関連するコンテンツはまとめ、視線誘導が自然になるように配置する。

## 階層
- 情報の優先順位を視覚的に明確に伝えるために、サイズ、太さ、余白、色などを用いる。
- 最も重要な情報や操作は、画面上で目立つ位置に配置する。

## 明瞭性
- 過度に複雑なレイアウトや、情報過多を避ける。
- 必要な情報にすぐアクセスできるように、表示内容の簡潔化と構造の明確化を図る。

## 応答性（レスポンシブデザイン）
- さまざまな画面サイズや向き（Portrait/Landscape）に適応するように設計する。
- Auto LayoutやSize Classesなどのシステム提供ツールを活用する。
- iPadやMacなど複数デバイス間でも自然にスケーリングできる設計が望ましい。

## プラットフォーム別考慮点

### iOS・iPadOS
- Safe Areaの尊重：ノッチやホームインジケータを避けて表示要素を配置。
- Touch Targetの確保：少なくとも44pt × 44ptを推奨。
- Spacing Guides：システム提供のスペースガイドライン（例：標準のマージン）を活用。

### macOS
- 複雑な情報を提示しやすいため、グリッドベースで整理された密度の高いレイアウトが可能。
- ウィンドウサイズが可変であることを前提に、レイアウトが柔軟に変化する設計が必要。

### watchOS
- 画面サイズが非常に小さいため、縦方向のシンプルなレイアウトを採用。
- タップしやすい大きな要素の使用が推奨される。

### tvOS
- 大画面に適したスケーラブルなレイアウトが求められる。
- フォーカス駆動のUI（リモコン操作）に対応したレイアウト構造を採用する。

## 重要ガイドライン
- 余白やパディングの設定は、ユーザーがUIを素早く理解する上で重要な要素。
- コンテンツは視覚的に自然なグルーピングを行い、セクションや階層構造が認識しやすいように設計する。
- レイアウトとタイポグラフィ、カラー、アイコンのバランスを総合的に整えることで、明快で使いやすいUIが実現できる。
